1. Recommendation pour l'environnement de production

    Par défaut utilise la base de données Apache Derby. Dans un environnement de production vous devriez opter pour MySQL, Postgres, IBM DB2 ou Oracle.

  2. Activer l'upload et l'import d'images vers le tableau blanc

    • Installez ImageMagick sur le serveur. Vous trouverez plus d'informations à l'adresse http://www.imagemagick.org. Les instructions d'installation peuvent être trouvées ici http://www.imagemagick.org/script/binary-releases.php. Sur la majorité des systèmes linux vous pouvez le trouver via votre manager de paquets (apt-get).

  3. Activer l'import de fichiers PDF sur le tableau blanc

    • Installez GhostScript sur le serveur. Vous trouverez plus d'information sur http://pages.cs.wisc.edu/~ghost/. Sur la majorité des systèmes linux vous pouvez le trouver via votre manager de paquets (apt-get).
    • Installez SWFTools sur le serveur. Vous trouverez plus d'informations sur http://www.swftools.org/. Certaines distribution de Linux le possède déjà. Voir http://packages.debian.org/unstable/utils/swftools). La version recommandée de SWFTools est 0.9 - les versions précédentes ont un bug qui modifie les dimensions des objets sur le tableau blanc.

  4. Activer l'import de documents Office (.doc, .docx, .ppt, .pptx, etc.) sur le tableau blanc

    • Le service OpenOffice doit être démarré et en écoute sur le port 8100. Voir OpenOfficeConverter pour plus de détails.

  5. Activer l'enregistrement et l'import de fichiers vidéo (.avi, .flv, .mov und .mp4) sur le tableau blanc

    • Installez FFmpeg. Vous devez obtenir la dernière version de FFMPEG ! Pour Windows vous pouvez la télécharger par exemple de http://ffmpeg.arrozcru.org/builds/. Pour Linux ou OSx vous devriez utiliser l'une des nombreuses instructions d'installation sur le web. Vous devez activer libmp3lame !
    • Installez SoX http://sox.sourceforge.net/. Vous devez installer la dernière version ! SoX 12.xx ne fonctionnera PAS !
